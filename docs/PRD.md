# 產品需求文件 (PRD)：演唱會搶票系統

## 1. 產品概述
本系統旨在提供一個公平、穩定且高效的線上售票平台。透過現代化的架構設計，確保在熱門場次開賣瞬間，系統能處理數百萬次的請求，並有效防止黃牛腳本自動化搶票，提供真實樂迷順暢的購票體驗。

## 2. 目標用戶
* **一般樂迷：** 希望透過公平競爭、流暢介面買到心儀演唱會門票的人。
* **主辦單位：** 需要發布活動訊息、管理座位表、監控售票狀況的機構。
* **系統管理員：** 負責後台維運、異常訂單查核與金流退款處理。

---

## 3. 主要功能說明

### 3.1 實名制帳號與身分認證
為打擊黃牛，系統要求用戶在購票前完成手機與身分證件綁定。
* **功能描述：** 用戶註冊後需通過簡訊驗證 (OTP) 與實名資訊輸入。
* **技術重點：** 一組證件號碼僅能對應一個帳號，並在搶票當下限定帳號購票張數。

### 3.2 動態虛擬等候室 (Queue System)
在開賣瞬間將流量引導至緩衝隊列，避免資料庫崩潰。
* **功能描述：** 用戶點擊購票後進入等候頁面，顯示預估等待時間與當前排序。
* **技術重點：** 使用 Redis 處理高頻率排隊機制，防止用戶手動重新整理頁面導致排名遺失。

### 3.3 自動化防黃牛驗證機制
透過行為分析與圖形驗證，過濾掉自動化搶票腳本。
* **功能描述：** 在進入選位前，要求用戶輸入與藝人/演唱會相關的問答題，或執行滑動驗證。
* **技術重點：** AI 驅動的異常流量偵測（例如同一 IP 頻繁發出請求）。

### 3.4 即時座位點選與鎖定
視覺化場館地圖，讓用戶即時選位並暫時保留。
* **功能描述：** 用戶在座位表上點選後，系統會鎖定該座位 10 分鐘，期間內其他用戶無法選取。
* **技術重點：** 分散式鎖定機制（Distributed Lock），確保不會發生「重號（Double Booking）」問題。

### 3.5 多元金流與超時自動釋票
支援多種支付方式，並確保未結帳座位能快速回流市場。
* **功能描述：** 提供信用卡、第三方支付（如 LINE Pay、Apple Pay）及虛擬帳號匯款。
* **技術重點：** 嚴格的超時監控，一旦 10 分鐘內未完成付款流程，系統立即自動釋出座位。

## 4. 非功能性需求 (技術指標)
| 指標 | 要求 |
| :--- | :--- |
| **高併發處理** | 需支援開賣首秒 500,000 TPS 以上的吞吐量。 |
| **低延遲** | 核心購票 API 響應時間需小於 200ms。 |
| **可用性** | 系統年度可用性達 99.99% (High Availability)。 |
| **安全性** | 符合 PCI DSS 支付卡產業資料安全標準，保障個資。 |

---

## 5. 使用者流程 (User Flow)
1.  **預熱階段：** 用戶登入、完成實名驗證、收藏心儀場次。
2.  **開賣時刻：** 點擊搶票按鈕 -> 進入虛擬等候室排隊。
3.  **驗證階段：** 回答防黃牛問題 -> 通過後進入座位表。
4.  **選位階段：** 點擊空位 -> 系統成功鎖定座位。
5.  **結帳階段：** 填寫取票資訊 -> 跳轉金流支付 -> 訂單成立。

---

## 6. 風險評估
* **黃牛進化：** 需定期更新驗證題庫與偵測邏輯。
* **瞬間流量過載：** 需準備自動擴容（Auto-scaling）與 CDN 靜態資源分流。
* **金流跳轉失敗：** 需建立完善的對帳與回滾機制，確保座位不會因金流錯誤而卡死。
